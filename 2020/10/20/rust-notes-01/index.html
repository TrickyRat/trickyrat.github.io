Rust 01 Get Started

Rust的安装

在Rust官网下载相应平台的安装器进行安装。官方也有安装教程，这里就不赘述了。

cargo

cargo是Rust的包管理器，可以用来创建项目、安装依赖、类型检测、编译、运行以及测试项目等功能。cargo是一个功能强大的工具。

常用的cargo命令

命令 说明 示例
cargo new 创建项目，默认创建binary项目，添加参数可以选择lib和bin两种项目 cargo new hello --lib(默认–bin)
cargo check 类型检测 cargo check
cargo build 编译项目 cargo build [–release]
cargo run 运行项目，可以带参数 cargo run [–release]
cargo test 对项目进行单元测试 cargo test

详细的说明可以查看官方文档。

第一个rust项目

使用cargo new hello-world创建一个项目，然后cd hello-world进入项目目录后可以看到项目的结构

C:.
│  .gitignore
│  Cargo.toml
│
└─src
        main.rs

会有一个Cargo.toml和一个src\main.rs，其中Cargo.toml文件是负责配置cargo和项目依赖项，main.rs文件则是程序的入口点，里面的内容

fn main() {
    println!("Hello, world!");
}

简单易懂，关键字fn用来指明这是一个无返回值名称为main的function，然后函数体内调用rust的宏println!输出了Hello, world!

在Rust中所有的函数调用都是必需有返回值的表达式，无返回值的返回一个空的tuple表示或者省略不写。

fn foo() -> () {}
// 二者等效
fn foo() {}

而添加--release参数后，则会生成不带debug信息且优化后的代码，如下图所示

执行cargo build命令后会在项目目录下生成一个target文件夹，target文件夹中的内容就是编译生成的结果。根据cargo build后面添加的参数会生成两个文件夹，即release和debug。

也可以直接运行cargo run命令直接运行项目，这个命令就相当于cargo build然后在执行编译后的可执行文件。和cargo build一样默认是生成debug代码，带上--release参数后则是release代码。

依赖项

在项目文件夹中找到Cargo.toml文件，这个文件便是Rust项目的依赖项配置文件

在这个文件里对项目进行配置，比如我现在需要使用随机数，那么我只需要在[dependencies]下面添加rand = "0.7.3"，然后项目运行时，cargo会自己进行依赖还原，将详细的包写入Cargo.lock文件中

并且会自动去解析引入包的依赖，这和npm相似。
